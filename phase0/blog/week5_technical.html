When to Use Classes in Ruby

Date: April 12, 2014

A class in Ruby creates a template or format for which new objects can be created.

For example, if we were going to be listing a bunch of pets, say for a shelter. We'd probably want to create a class pet that lists name, breed, age, among other things.

class Pet

def initialize(name, breed, age)
@name = name
@breed = breed
@age = age
end

For each new pet object, we can just create it using the format that the class sets out for us.

Pet.new("Bobo", "corgi", 5)

You want to use classes when you are going to create different objects from the same format, or test/do something through the same criteria. In order to save on repition, you would use a class.

For example, in our dice exercise, we were rolling a new die each time, and we used a class so that bit of code was resuable each time we wanted to go ahead and roll the die.

Within the class, the "@" sign means that the variable in question is an instance variable, and in this case it is a class wide variable. That means that you can call on in anywhere within the class "Pet," even outside of the context of "initialize." This is important because we want to be able to reuse these variables in other methods within the class without having to create duplicate variables.

The cool thing about Classes is that you can define methods inside of them, both public and private. A public method is one that you can call from anywhere, and a private one is one that you can only call from inside/within the class. This creates a variety of options for programmers.
