Arrays vs. Hashes in Ruby

Date: March 29, 2014

Arrays in Ruby are a way of saving multiple values in one variable. The format is as follows

my_array_name = [1, 2, "string", 4, 5]

Arrays can store numbers, strings or even other arrays. You can access elements in the array with the following syntax, by using their index number, starting from 0:

my_array_name[0] # => 1
my_array_name[1] # => 2
my_array_name[2] # => "string"
my_array_name[3] # => 4
my_array_name[4] # => 5

Hashes on the other hand, are similar to Arrays in that they can store multiple values. But instead of using a numbered index to access values, hashes use key: value pairs. The notation looks something like this:

my_hash_name = { key: value }

Hashes can also store numbers, strings, other hashes and arrays. Similar to Arrays, you can access elements in a hash by referring to the key, as illustrated by the following example:

new_hash = { red: 1,
blue: 8,
yellow: 10}
new_hash[red] # => 1

While Hashes and Arrays are similar, they are not always substitutable. For example, if you were using a variable to hold a list of people's names and test scores, you would probably want to use a Hash. That way you could access each person's test score by using their name as a key. Otherwise, accessing each score using a numbered index would make less sense, and you wouldn't be able to link each person to their scores.

test scores as a hash

test_scores = {
adam : 90,
bob : 75,
cathy: 89
}

test scores as an array

test_scores = [90, 75, 89]

Take another example: If you were using a variable to hold a list of desserts, you probably wouldn't need to use a hash because there wouldn't be a need for key value pairs. You coul just structure it as follows:

desserts = ["strawverry chiffon cake", "apple pie", "lemon sorbet"]

When coding it is best to think about the intended usage and structure of the code before implementing the data structure that fits, whether it be a hash, array, or something else entirely.
